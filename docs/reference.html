Element reference

Alarm source

The `Source` element identifies where the alarm originates. The draft allows either direct text content or a structured set of child elements.

Element Description Required
ClientClient, company, or customer identifier.No
DepartmentDepartment within the client's organization.No
BuildingBuilding identifier when there are multiple buildings.No
WardWard, for example in a hospital.No
SectionSection of the building.No
RoomRoom number.No
DetectorDetector identifier.No
TransmitterWireless transmitter identifier or serial number.No
PhonePhone number.No
AddressAddress in another addressing scheme such as a bus address, IP address, or email address.No
 
<Source>219</Source>
<Source>
    <Client>10026</Client>
    <Building>B</Building>
    <Ward>2</Ward>
    <Room>219</Room>
</Source>

Element reference

Alarm type

The `Type` element carries the alarm type identifier as its text content. The draft also defines two optional attributes.

Attribute Description Required
QualifierQualifier such as `New`, `Updated`, or `Reset`, as used in the draft examples.No
SeveritySeverity value from 0 to 100, where 100 is most severe.No

Element reference

Alarm location

The `Location` element is optional and can describe the alarm position through a beacon identifier, cell information, or GPS data.

Element Description Required
CellInfoMobile network cell information.No
BeaconIdBeacon identifier as direct content.No
GpsGPS coordinates and related metadata.No

CellInfo

Element Description Required
MCCMobile country code, 3 digits.Yes
MNCMobile network code.Yes
LACLocation area code, represented in decimal.Yes
CICell identifier, represented in decimal.Yes
TATiming advance.No

Gps

Element Description Required
LongitudeLongitude in decimal degrees. Negative values indicate west of the prime meridian.No
LatitudeLatitude in decimal degrees.No
AltitudeAltitude in metres. Negative values are allowed below sea level.No
DirectionNot described further in the published draft.No
SpeedNot described further in the published draft.No
TimeStampNot described further in the published draft.No
HorizontalAccuracyNot described further in the published draft.No
VerticalAccuracyNot described further in the published draft.No
DirectionAccuracyNot described further in the published draft.No
SpeedAccuracyNot described further in the published draft.No

    </CellInfo>
</Location>

Appendix

Result codes

Code Default text Description
200OKMessage was accepted or delivered and acknowledged by the final destination.
202AcceptedMessage was accepted and queued, but not yet delivered.
400Bad RequestThe datagram was malformed.
401UnauthorizedIdentification or authorization is required.
403ForbiddenThe client lacks sufficient privileges for the requested action.
404Not FoundThe destination was not found.
500Internal Server ErrorThe server failed while handling the message.
503Service UnavailableThe server or an external dependency is temporarily unavailable.
