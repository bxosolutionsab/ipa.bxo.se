Overview

A smaller, easier path through the draft

This site keeps the current OpenIPA draft in a static form, but reorganizes it into a short documentation set that is easier to maintain and easier to read. The goal is not to expand the protocol, but to make the published material clearer and more internally consistent.

What OpenIPA is for

OpenIPA is designed for alarm and message delivery over modern IP networks, while staying simple enough for practical integration work.

What changed in this rewrite

The draft is now split into focused pages, message names and acknowledgement rules are phrased consistently, and examples now match the documented field names.

What still remains a draft

Some fields are clearly defined, while others are only sketched. Where the published draft is underspecified, this site says so instead of inventing behavior.

At a glance

Implementation notes

Transport

Messages are XML payloads framed over a TCP transport with a fixed `PUT / IPA/1.0` request line and a mandatory `Content-Length` header.

Encoding

The current draft says message encoding is UTF-8 and also allows pure 7-bit ASCII.

Acknowledgements

If a message carries an `Id`, the receiver is expected to reply with an `Ack` that copies that `Id` and returns a `Result` code.

Scope

The published message set currently covers `Ack`, `Heartbeat`, `TextMessage`, and `Alarm`.
