Overview

Internet Protocol for Alarm Management

IPA (Internet Protocol for Alarm management) is designed to replace the widely used ESPA 4.4.4 standard. ESPA defines how to send text messages over serial links like RS-232 to pagers. The purpose of IPA is to provide a similarly simple, easy to use, reliable and open standard that is suitable for transmission over the now abundant IP infrastructure, including the Internet and local area networks.

What OpenIPA is for

OpenIPA is intended for alarm and message delivery over IP networks while preserving the simple integration model that made ESPA practical to deploy.

Why it exists

It provides an open protocol for modern network infrastructure without losing the ease of use and reliability expected from alarm-management integrations.

Current status

The specification is still being worked on, and this site reflects the current draft of the first version of the emerging protocol.

At a glance

Implementation notes

Transport

Messages are XML payloads framed over a TCP transport with a fixed `PUT / IPA/1.0` request line and a mandatory `Content-Length` header.

Encoding

The current draft says message encoding is UTF-8 and also allows pure 7-bit ASCII.

Acknowledgements

If a message carries an `Id`, the receiver is expected to reply with an `Ack` that copies that `Id` and returns a `Result` code.

Scope

The published message set currently covers `Ack`, `Heartbeat`, `TextMessage`, and `Alarm`.
